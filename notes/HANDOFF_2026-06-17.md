## Working setup (all built-in — the custom serial_proxy is retired)
- **Text/boot log:** `pio device monitor -e esp32dev-mini --port COM6` → writes
  `logs/device-monitor-*.log` (log2file). Read the newest.
- **Live frames (the RE instrument):** device `/wire` page (WebSocket `/canstream`, carries
  `@RX` + `@TX`). Or `tools/ws_capture.py <ip> <out.canlog> <secs>` for a lossless file.
- **Drive the display:** device `/preview` page, or the HTTP routes directly.
- **Decode a capture:** `tools/affa_decode.py LOG --id <hex> [--merge]` (merge unions lossy
  fragments of a static screen).
- **Log verbosity (runtime):** `GET /api/loglevel?n=0..4` (E/W/I/D/T). Default INF is clean;
  `n=4` (TRC) shows per-frame `[CAN] RX/TX` dumps. HTTP is the ONLY knob — **serial INPUT is
  dead** on the C3 USB-CDC (HWCDC RX broken on Windows; documented in `SerialWireLink.h`).
- **Transport route:** must be `both` (or `can`) to drive the REAL panel —
  `GET /api/route?mode=both`. `virtual` cuts CAN TX (bench-only). `/preview` now defaults to
  `both` and shows the active route.

## Key reverse-engineering findings (CAN id map, Carminat)
- **0x151** text/menu/popup screens · ACK **0x551**.
- **0x1C1** key events · ACK **0x5C1**. Password = **5-3-2-1** (`sendPasswordSequence`), confirmed
  auto-entering on the CODE screen.
- **0x1F1** = **navigation channel (AFFA3NAV)** · ACK **0x5F1**. 302-byte ISO-TP message. The
  "earth/planet" is the **idle** state (no route); when routing it should carry turn arrow +
  distance + street. Treat it as **structured fields, not a bitmap**.
- **CODE/password screen** = windowed text `77 55 55 FF 60 01`: shows "CODE" then "`<B0>`000",
  `0xB0` = blink cursor advancing per digit.
- **Popup overlay** = `0x151 10 0E 74 <icon> 55 <srcIcon> <fmt> 01` + ≤8 text bytes. mode 0x74.
  Non-destructive (screen redraws under it). Panel renders **8 chars** max. icons cycle; known:
  `09`=volume ctx, `94`=empty, `9B`=traffic. See `notes/AFFA3_ICONS.md` / `AFFA3_SCREENS.md`.

## Gotchas learned
- Multiple `pio device monitor` instances pile up and fight over COM6 (PermissionError) — kill
  stray `python.exe`/miniterm before starting one. Don't background the monitor with bash `&`
  (dies with the shell); use the harness background runner.
- After any flash the device reboots → route resets to `both`, log level to INF (defaults).
